# What Is Programming? 

## Code Gives Instructions to a Machine

Programming is writing instructions a computer follows exactly, in order, without interpretation or improvisation.

Think of a recipe:

1. Heat oven to 180C
2. Mix flour, sugar, eggs
3. Pour into pan
4. Bake for 25 minutes

A recipe has a clear sequence. Each step depends on the previous one. If you skip step 1, step 4 fails. Programming works the same way.

## What a Program Does

Every program, regardless of language, does three things:

1. **Takes input** -- from a user, a file, a network request, a sensor
2. **Processes it** -- calculations, decisions, transformations
3. **Produces output** -- displays a result, saves data, sends a response

A web application follows this pattern at every level. A user submits a form (input). The server validates and stores the data (processing). The browser shows a confirmation page (output).

## Why This Matters for Rails

Rails is a framework for building web applications. A web application is a program that takes HTTP requests as input, processes them with business logic, and produces HTML or JSON as output.

When you write Rails code, you are writing instructions for what should happen when a user visits a URL. That is all it is. The framework handles the repetitive plumbing so you focus on the logic specific to your application.

## Key Terms

- **Source code** -- The instructions you write in a programming language
- **Execution** -- The computer following those instructions
- **Runtime** -- The period when the program is executing
- **Bug** -- An instruction that does the wrong thing, or the right thing in the wrong order

## Practical Takeaway

You do not need to understand computers deeply to program. You need to be precise about what you want to happen and in what order. The computer handles the rest. If your instructions are wrong, the result is wrong. If your instructions are right, the result is right. There is no in-between.
